Question 3
In a RESTful API, which HTTP method is most appropriate for partially updating an existing resource without replacing the entire object?
A GET — retrieves the current state of the resource
B PUT — replaces the entire resource with new data
C PATCH — applies a partial modification to the resource
D POST — creates a new resource at the endpoint
✦ AI Explanation

Correct! PATCH is designed for partial updates — only the changed fields are sent. PUT replaces the entire resource. This distinction is key for resource-efficient API design, covered in Lecture 7 slide 12.
